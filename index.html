    Fonctionnement de l'API

    L'API répresent les données qui sont nécessaires pour remplir la galerie de photos à l'aide d'AJAX. Tous les méthodes sont des actions GET.

    URIs de l'API

    Action URI Description
    GET /photos Obtenir toutes la collection de photos
    GET /photos/:id Obtenir toutes les informations d’une photo
    GET /especes/id:/photos Obtenir toutes les photos d’une espèce
    GET /familles/:id/photos Obtenir toutes les photos d’une famille
    GET /ordres/:id/photos Obtenir toutes les photos d’une ordre
    GET /auteurs/:id/photos Obtenir toutes les photos d’un photographe/auteur
    GET /especes Obtenir les informations d’une espèce

    GET /ordres/:id Obtenir les informations d’un ordre
    GET /auteurs Obtenir toutes les auteurs/photographes

    Paramètres dans l'URI

    L'API fait une pagination de 30 éléments par page par défaut; ce nombre d'éléments peut être modifié en ajoutant le pàrametre "elements=" : 

    - /photos?elements=10

    Pour changer à la page suivante des éléments, vous pouvez utiliser le paramètre "page=". 

    - /photos?page=2
    - /photos?elements=20&page=5

    Par défaut, pour le storie concernant aux photos, les éléments sont triés par le nom de l'espèce, mais si vous voulez les trier par an autre critère, il suffit d'ajouter le para mètre "trier=" : 

    - /photos?trier=Num_Ph
    - /photos?elements=3&trier=Num_Ph

    Pour le filtrage de données concernants aux stories des photos, la requète WHERE de est construite avec les colonnes de la base de données plus la valeur que l'on veut obtenir : 

    - /photos?Sexe=0&Espece_S_Acc=XXXX&...
